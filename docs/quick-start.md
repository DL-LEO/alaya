# Alaya · Quick Start Guide

## The Goal

Alaya is not just another knowledge base tool. Its purpose is **转识成智** — transforming stored consciousness (识, raw knowledge) into living wisdom (智, multi-perspective understanding).

In Yogacara Buddhism, this transformation happens through the Eight Consciousnesses:
- **Alaya** (阿赖耶识) stores the seeds → your knowledge base
- **Manas** (末那识) grasps from a fixed perspective → each persona's unique lens
- **Sixth Consciousness** (第六意识) discriminates → LLM reasoning in persona's voice

Through **xunxi** (熏习, cultivation), every interaction deepens the transformation. Knowledge that was flat becomes layered. What was stored becomes lived.

### 2. Initialize

Just say:

> **"alaya init"** or **"启用识海"**

The Agent will automatically:
- Ask you where your knowledge base lives (with option to use Obsidian vault)
- Set up the `alaya/` directory with default config
- Create `raw/` directory for source documents
- Copy 8 default personas (Manas) with character profiles
- Guide you through next steps after setup

> **💡 Tip**: If you use [Obsidian](https://obsidian.md), choose your Obsidian vault path
> during setup — your wiki cards will appear in Obsidian's Graph View automatically.

Organize your knowledge into category subfolders:

│   ├── deep-learning_category.md    ← Auto-generated by "build index"
│   ├── Transformer-Architecture.md
│   └── Neural-Networks.md
├── fault-diagnosis/
│   ├── fault-diagnosis_category.md
│   └── GNN-Fault-Detection.md
└── philosophy/
    ├── philosophy_category.md

Then say: **"build index"** — this generates the three-layer knowledge graph.

Try asking a question with a persona:

> "Feynman, what do you think about attention mechanisms?"

Then ask the same question to a different persona:

> "Socrates, what's YOUR take on attention?"

> "Buddha, how does attention relate to consciousness?"

**Same knowledge base. Three completely different perspectives.** This IS the Manas at work — each persona selects and interprets different cards from the same candidate pool. And every interaction leaves traces (xunxi), gradually transforming raw knowledge into multi-perspective wisdom.

## Three-System Architecture

Alaya consists of three independent subsystems:

| System | Data Directory | What It Does |
|:--|:--|:--|
| **Knowledge** | `wiki/` | Description-driven knowledge graph (index → category → card) |
| **Memory** | `alaya/memory/` | Per-persona interaction history + shared mood/attention state |
| **Persona** | `alaya/manas/` | Persona definitions, affinity, voice profiles |

Each system has its own config section (`config.knowledge`/`memory`/`persona`) with independent version numbers. Updates to one system don't affect the others' data.

### Description-Driven Knowledge Graph

```
Layer 1 · wiki/index.md (~1-2KB)
  Category wikilinks + description paragraphs
  LLM reads this to match user questions against categories

Layer 2 · wiki/{category}/{category}_category.md
  Category header description + ## Cards (name + one-line description)
  LLM reads only the ## Cards section to build candidate pool

Layer 3 · wiki/{category}/*.md
  Individual knowledge cards with description field in YAML
  Full content loaded only for persona-selected cards
```

**v2.0**: No explicit graph edges, no tag computation. Card descriptions and category headers form a natural-language "graph" — LLM semantic matching discovers connections that explicit links would miss. Cross-category relationships are expressed in prose within descriptions.

### Emotional Memory System

Each persona builds unique relationship memories through interaction:

```
alaya/memory/
├── {persona}_history.json   ← Per-persona interaction history (hot/cold zones)
└── ambient.json             ← Shared mood + attention state (cross-persona)
```

- **Per-persona history**: each persona only sees their own conversations — preserves immersion
- **Shared ambient state**: all personas sense user's recent mood and attention focus
- **Warm Recall**: structured data is transformed into natural, character-voiced memories at read time

This means Feynman naturally says "You were curious about transformers last time" while 小昭 says "公子最近好认真呀" — from the same system, each in their own voice.
