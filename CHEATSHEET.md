# Alaya 识海 · 常用命令

---

## 🔍 检索与索引

| 触发词 | 功能说明 | 什么时候用 |
|:--|:--|:--|
| `"同步索引"` / `"构建索引"` / `"sync index"` / `"build index"` | **快速同步，无需 LLM**。直接执行 `build_index.py --full`，用各分类 refined 块重新生成 `index.md` base 块。同时补充缺失卡片描述。**不会覆盖分类 refined 和 MANUAL 内容。** | ✓ BI 提示索引不同步<br>✓ 新增/重命名卡片后<br>✓ 日常快速重建 |
| `"补充描述"` / `"补充描述 世界模型"` / `"补充描述 {card_name}"` | **自动补全卡片描述**。扫描缺少 description 字段的卡片，从正文提取（优先级：摘要段 → 贡献句 → blockquote → 首段落）。支持三种粒度：<br>① **无参数**：扫描所有分类所有卡片<br>② **指定分类**：如 `"补充描述 世界模型"`，仅处理该分类<br>③ **指定单卡**：如 `"补充描述 Transformer"`，仅处理一张卡片 | ✓ 导入新卡片后发现描述缺失<br>✓ 健康检查报告提示缺失 |
| `"更新类别描述"` / `"更新类别描述 世界模型"` | **LLM 精化分类头部**。读取某分类所有卡片的 description，生成 150-300 字精化概览，写入 `<!-- AUTO (refined) -->` 块。支持指定单一分类。精化内容三段结构：领域定位 → 核心议题 → 阅读指引，并自然引用交叉分类。 | ✓ 新分类导入后首次精化<br>✓ 分类新增大量卡片后<br>✓ BI 检测到描述陈旧 |
| `"更新索引描述"` / `"更新索引描述 世界模型"` | **LLM 精化所有分类 refined 块 + 同步至索引**。先精化各分类 refined 块（单数据源），再自动执行 `--full` 同步到 `index.md` 的 base 块。一个命令搞定全部，无需再记脚本。 | ✓ 想要整体提升描述质量<br>✓ 新分类首次注册<br>✓ 旧描述风格不统一 |

> **推荐顺序**：`同步索引`（日常） → `更新类别描述`（精化个别分类） → `更新索引描述`（全面精化）
>
> 💡 **日常用 `同步索引`**，5 秒完成；**隔几周用一次 `更新索引描述`**，让 LLM 重新精化描述质量

## 👤 角色

| 触发词 | 功能说明 | 什么时候用 |
|:--|:--|:--|
| `"创建角色"` / `"蒸馏角色"` / `"distill persona"` | **7 阶段角色创建协议**。定义定位 → 提取对话风格 → 梳理关怀维度 → 建立知识边界 → 确定触发场景 → 设计签名 → 生成 JSON + profile.md。 | ✓ 需要特定专业知识的新角色<br>✓ 从某本书/人物提炼角色 |
| `"克隆 {name}"` / `"clone {name}"` | 克隆现有角色所有配置，进入定制流程修改差异部分。 | ✓ 想基于已有角色做微调 |
| `"删除角色 {name}"` / `"delete persona {name}"` | 删除角色的所有文件（JSON + profile.md + history）。**不可逆操作。** | ✓ 角色不再需要 |
| `"各位大佬"` / `"识海群聊"` / `"一起群聊"` / `"开个群聊"` / `"群聊模式"` / `"大家来聊聊"` | **多角色群组讨论**。参与角色轮流发言，保留各自视角和风格。 | ✓ 复杂问题需多视角分析<br>✓ 教学场景需多位名家对话 |
| `"显示角色"` / `"list personas"` / `"有哪些角色"` | 列出所有可用角色：定位、专长、兴趣焦点、签名句式。 | ✓ 记不清有哪些角色 |
| `"{角色名}, {问题}"`（如 `"Feynman, 什么是量子纠缠？"`） | 强制指定某个角色回答问题，绕过自动选择。 | ✓ 明确知道想让谁回答 |

## 📥 导入

| 触发词 | 功能说明 | 什么时候用 |
|:--|:--|:--|
| `"导入论文 {url/文件路径}"` / `"import paper {url}"` | **将论文导入知识库**。三种模式：**info**（基础信息+摘要，省时）、**full**（完整结构化提取含方法/实验/结论，深入）、**extract**（内容精要提取，折中）。自动创建知识卡片并注入元数据。 | ✓ 看到想收藏的论文<br>✓ 阅读文献需做笔记 |
| `"批量导入 {path}"` / `"batch import {path}"` | **批量导入多篇文件**。三种模式：① **快速脚本**（批量基本信息提取，省时）② **并行导入PDF**（多进程并行，适合大批量）③ **LLM 智能**（逐篇深度结构化提取，质量最高）。 | ✓ 整个文件夹的论文<br>✓ 多篇同主题文献 |
| `"快速导入"` / `"并行导入PDF"` / `"深度导入论文"` / `"LLM导入"` | 快捷命令直接选择特定模式，绕过交互式模式选择。 | ✓ 明确知道需哪种模式 |
| `"深读 {card_name}"` | **回查卡片关联的原始文献内容**（如果导入时保留了原文）。比卡片摘要更详细。 | ✓ 卡片内容不够详细 |

| 触发词 | 功能说明 | 什么时候用 |
|:--|:--|:--|

## 🔧 维护

| 触发词 | 功能说明 | 什么时候用 |
|:--|:--|:--|
| `"健康检查"` / `"health check"` | 全系统健康检查：三层索引完整性、角色配置、元数据覆盖率、链接有效性、文件编码、系统文件状态。输出评分 + 建议操作。 | ✓ 每周例行维护<br>✓ 系统异常时<br>✓ 大规模导入/修改后 |
| `"运行熏习"` / `"run xunxi"` | **Level 2 熏习循环**。卡片强度衰减/活跃度更新、标签权重衰减、角色好感度衰减/遗忘曲线、情绪均值偏移。保持系统动态平衡。 | ✓ 话题切换时（自动检测）<br>✓ 手动触发维护<br>✓ 距上次熏习超 10 轮 |
| `"修复链接"` / `"fix links"` | 扫描修复 wikilink 问题：大小写不匹配、指向不存在卡片、循环引用检测。 | ✓ 健康检查报告链接问题<br>✓ 移动/重命名文件后 |
| `"BI报告"` / `"BI观察"` / `"天道观察"` | 生成 BI 观察报告：好感网络动态（角色间互信变化）、休眠角色提醒（>14 天未活跃）、知识缺口（兴趣 vs 卡片覆盖）。 | ✓ 每月例行分析<br>✓ 决策前查看系统状态 |
| `"查看配置"` / `"show config"` | 以可读格式显示当前系统参数（top_K、min_pool、半衰期、BI 开关等）。 | ✓ 了解当前参数<br>✓ 修改前先查看 |
| `"把 top_K 改成 5"` / `"关闭 BI"` | 自然语言修改配置项。支持：top_K、min_pool、half_life、bi_enabled 等。 | ✓ 调整检索参数 |
| `"修改配置"` | 进入配置修改引导模式。 | ✓ 需要多参数调整时 |

## 📌 推荐工作流

| 场景 | 步骤 |
|:--|:--|
| **日常维护** | `健康检查` → `同步索引`（按需） |
| **新增分类/大量导入后** | `同步索引` → `更新类别描述`（精化新分类） → `同步索引` → `健康检查` |
| **描述质量提升** | `同步索引`（先拉取最终卡片） → `更新索引描述`（LLM 全局精化） → `健康检查` |
| **系统异常** | `健康检查` → `修复链接` → `构建索引` → `健康检查` 验证 |

## 💡 小技巧

1. **大部分命令可指定单一分类**：如 `"更新索引描述 世界模型"`，避免全量 LLM 调用
2. **角色名可直接当命令用**：`"Feynman, 解释一下"` 强制指定角色
3. **`同步索引` 安全无副作用**：不会覆盖分类的 `<!-- AUTO (refined) -->` 和 `<!-- MANUAL -->` 内容
4. **每日用 `同步索引`，每月用 `更新索引描述`**：前者秒级完成，后者让 LLM 整体优化描述一致性
